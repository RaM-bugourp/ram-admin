# ram-adminx P0 权限修复 — 角色级访问控制

**日期**: 2026-06-09
**优先级**: P0

---

## 问题

1. 所有认证用户均可执行写操作，普通 `user` 角色可删除用户/角色
2. `is_superuser` 字段未被用于权限判断

## 修复方案

新增 `apps/rbac/permissions.py` — `IsAdminOrReadOnly` 权限类。

### 权限判断链

```
request.method in SAFE_METHODS?  →  放行（任何已认证用户）
is_superuser?                    →  放行
持有 root 角色?                  →  放行
持有 boss 角色?                  →  放行
其他                            →  403 Forbidden
```

### 改动文件

| 文件 | 改动 |
|------|------|
| `apps/rbac/permissions.py` | **新建** (42行) |
| `apps/rbac/views/user_views.py` | `permission_classes` 追加 `IsAdminOrReadOnly` |
| `apps/rbac/views/role_views.py` | `permission_classes` 追加 `IsAdminOrReadOnly` |

### 测试结果（8/8 通过）

| 角色 | GET | POST | PUT | DELETE |
|------|-----|------|-----|--------|
| boss | ✅ 200 | ✅ 201 | ✅ 200 | ✅ 204 |
| user | ✅ 200 | ❌ 403 | ❌ 403 | ❌ 403 |

# 2026-06-09 会话总结：ram-adminx v0.3.0 更新

## 今日完成的全部改动

### P0 — 权限系统
- **IsAdminOrReadOnly** 权限类 (42行)：普通 user 只读，root/boss 可写
- 应用到 `user_views.py` 和 `role_views.py` 的 `permission_classes`
- 8/8 测试通过

### P0 — 三级角色体系 (root/user/BOSS)
- **Role 模型**：新增 `is_unique` 布尔字段，`RolePermission` 预留表（0003迁移）
- **RoleService**：`enforce_unique_role()` BOSS 唯一性校验
- **UserService**：`create_user`/`update_user` 集成 BOSS 校验
- **RoleSerializer**：create/update/output 三套分离
- **RoleViewSet**：完整 CRUD（list/create/retrieve/update/destroy）
- **角色路由**：`urls_role.py` → 挂载到 `config/urls.py`
- **init_rbac**：自动创建 root/user/boss 角色 + admin(boss001/user001) 测试账号

### P1 — 前端功能补全
- **Dashboard 实时数据**：新建 `dashboard_views.py` + `dashboard.ts`，接入 `/api/dashboard/stats/`
- **前端按钮按角色显隐**：`isAdmin` getter + `v-if` 控制 UserListView/RoleListView 写按钮

### P2 — 后端防御性修复
- **软删除用户登录**：`backends.py` 加 `is_deleted=False` → 401
- **用户列表过滤**：`user_service.py` 加 `is_deleted=False`
- **last_login 冲突**：移除 `last_login_at` 手动赋值，用 Django 内置 `login()` 信号
- **Auth 接口返回角色**：login 和 user-info 均返回 `roles[]`
- **前端 Store**：新增 `roles/RoleInfo[]`、`isAdmin` getter、`roleCodes` getter

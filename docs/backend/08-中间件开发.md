# 08 - 中间件开发

## 适用场景
需要横切关注点时创建自定义中间件：请求日志、认证、事务管理、限流。

## 关联原则
- 原则3：关注点分离 — 中间件处理横切问题，不碰业务逻辑
- 原则10：结构化日志 — 请求日志输出 JSON 格式

## 核心规则
1. **中间件顺序敏感** — CORS 在最前，事务在最后
2. **中间件不包含业务逻辑** — 只做通用横切
3. **异常友好** — 中间件异常不能吞掉业务异常

## 推荐中间件顺序

```python
MIDDLEWARE = [
    'corsheaders.middleware.CorsMiddleware',          # 1. CORS 最前
```

## 开发步骤

### 1. RequestId 中间件 — 全链路追踪

文件：`apps/audit/middleware.py`

```python
import uuid
import time
import logging

logger = logging.getLogger('audit.request')

class RequestIdMiddleware:
    """为每个请求注入唯一 ID，用于日志关联"""

    def __init__(self, get_response):
        self.get_response = get_response

    def __call__(self, request):
        request.request_id = str(uuid.uuid4())[:8]
        response = self.get_response(request)
        response['X-Request-ID'] = request.request_id
        return response

class RequestLogMiddleware:
    """记录请求信息（JSON 格式）"""

    def __init__(self, get_response):
        self.get_response = get_response

    def __call__(self, request):
        logger.info("Request start", extra={
            'request_id': getattr(request, 'request_id', '-'),
            'method': request.method,
            'path': request.path,
            'user_id': getattr(request.user, 'id', None),
            'ip': self._get_client_ip(request),
        })
        response = self.get_response(request)
        logger.info("Request end", extra={
            'request_id': getattr(request, 'request_id', '-'),
            'status': response.status_code,
        })
        return response

    @staticmethod
    def _get_client_ip(request):
        x_forwarded = request.META.get('HTTP_X_FORWARDED_FOR')
        return x_forwarded.split(',')[0].strip() if x_forwarded else request.META.get('REMOTE_ADDR')

class RequestDurationMiddleware:
    """计算请求耗时"""

    def __init__(self, get_response):
        self.get_response = get_response

    def __call__(self, request):
        start = time.monotonic()
        response = self.get_response(request)
        duration_ms = (time.monotonic() - start) * 1000
        response['X-Response-Time'] = f"{duration_ms:.2f}ms"
        return response
```

### 2. 限流中间件（可选）

```python
from django.core.cache import cache
from django.http import JsonResponse

class RateLimitMiddleware:
    """简单 IP 限流"""

    def __init__(self, get_response, max_requests=100, window_sec=60):
        self.get_response = get_response
        self.max_requests = max_requests
        self.window_sec = window_sec

    def __call__(self, request):
        if request.path.startswith('/api/'):
            ip = request.META.get('REMOTE_ADDR', 'unknown')
            cache_key = f'rate_limit:{ip}'
            count = cache.get(cache_key, 0)

            if count >= self.max_requests:
                return JsonResponse(
                    {'error': {'code': 'RATE_LIMITED', 'message': '请求过于频繁'}},
                    status=429
                )

            cache.set(cache_key, count + 1, self.window_sec)
        return self.get_response(request)
```

## 反模式（禁止）
- ❌ 中间件里写业务逻辑
- ❌ 中间件顺序混乱（CORS 不在最前）
- ❌ 中间件 catch Exception 并吞掉
- ❌ 日志用 print 而不是 logger

## 自检清单
- [ ] 中间件顺序正确？（CORS → 安全 → 日志 → 认证 → 事务）

- [ ] 响应头包含 X-Request-ID？

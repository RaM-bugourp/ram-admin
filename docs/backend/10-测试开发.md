# 10 - 测试开发

## 适用场景
所有功能开发完成后，按测试金字塔编写测试。

## 关联原则
- 原则8：测试驱动 — 先写测试，后写实现；单元 70% + 集成 20% + E2E 10%

## 核心规则
1. **测试金字塔**：单元 > 集成 > E2E（70/20/10）
2. **TDD 流程**：AI 先生成测试 → 代码 → 运行测试
3. **Mock 外部依赖**（数据库/缓存/外部 API）
4. **测试不依赖执行顺序**

### 步骤 2：单元测试 — Service 层

文件：`backend/tests/unit/test_user_service.py`

```python
import pytest
from unittest.mock import Mock
from apps.rbac.services.user_service import UserService
from apps.common.exceptions import BusinessError

class TestCreateUser:
    """用户创建 - 单元测试"""

    def test_create_user_success(self):
        mock_repo = Mock()
        mock_repo.get_by_email.return_value = None
        mock_repo.create.return_value = Mock(id=1, username='test', email='test@test.com')

        service = UserService(user_repo=mock_repo)
        result = service.create_user({
            'username': 'test',
            'email': 'test@test.com',
            'password': 'Pass1234',
            'role_codes': []
        })

        assert result['username'] == 'test'
        mock_repo.create.assert_called_once()

    def test_admin_requires_department(self):
        mock_repo = Mock()
        service = UserService(user_repo=mock_repo)

        with pytest.raises(BusinessError) as exc:
            service.create_user({
                'username': 'admin',
                'email': 'admin@test.com',
                'password': 'Pass1234',
                'role_codes': ['admin'],
                'department_id': None
            })

        assert exc.value.code == 'ADMIN_REQUIRES_DEPARTMENT'

    @pytest.mark.parametrize("username,is_valid", [
        ("valid_user", True),
        ("user123", True),
        ("用户", True),
        ("invalid@user", False),
        ("", False),
    ])
    def test_username_validation(self, username, is_valid):
        assert UserService.is_valid_username(username) == is_valid
```

### 步骤 3：集成测试 — API 层

文件：`backend/tests/integration/test_user_api.py`

```python
import pytest
from django.urls import reverse

class TestUserAPI:
    """用户 API 集成测试"""

    def test_list_users_authenticated(self, auth_client, regular_user):
        url = reverse('user-list')
        response = auth_client.get(url)

        assert response.status_code == 200
        assert 'data' in response.json()

    def test_list_users_unauthenticated(self, api_client):
        url = reverse('user-list')
        response = api_client.get(url)

        assert response.status_code == 401

    def test_create_user_as_admin(self, auth_client):
        url = reverse('user-list')
        data = {
            'username': 'newuser',
            'email': 'new@test.com',
            'password': 'Pass1234',
        }
        response = auth_client.post(url, data)

        assert response.status_code == 201
        assert response.json()['data']['username'] == 'newuser'

    def test_create_user_duplicate_email(self, auth_client, regular_user):
        url = reverse('user-list')
        data = {
            'username': 'another',
            'email': regular_user.email,  # 重复邮箱
            'password': 'Pass1234',
        }
        response = auth_client.post(url, data)

        assert response.status_code == 400
        assert 'error' in response.json()
```

### 步骤 4：运行测试命令


## 测试用例设计原则
| 测试类型 | 被测对象 | Mock 什么 | 断言什么 |
|---------|---------|----------|---------|
| 单元测试 | Service 类 | Repository | 业务逻辑、异常 |
| 集成测试 | API 端点 | 无（用测试 DB） | 状态码、响应结构 |
| E2E 测试 | 完整流程 | 无 | 业务流程正确性 |

## 反模式（禁止）
- ❌ 测试之间互相依赖（依赖执行顺序）
- ❌ 单元测试连接真实数据库
- ❌ `assert True` 或空测试
- ❌ Mock 过度（连测试目标也 Mock 了）

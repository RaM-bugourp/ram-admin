# 23 - 可观测性与监控

## 适用场景
生产环境部署后，监控服务健康状态、追踪错误、指标采集。

## 关联原则
- 原则10：结构化日志 — JSON 格式便于日志采集系统解析
- 原则9：强制审查 — CI/CD 中集成健康检查

## 核心规则
1. **健康检查端点**：`/api/health/` 返回数据库/Redis/Celery 连通性
2. **Sentry 错误追踪**：自动捕获未处理异常
3. **JSON 日志**：标准输出，由 Fluentd/Logstash 采集
4. **关键指标**：请求QPS/延迟/错误率

## 开发步骤

### 步骤 1：健康检查端点

文件：`apps/common/views/health.py`

```python
from django.db import connections
from django_redis import get_redis_connection
from rest_framework import viewsets
from rest_framework.response import Response


class HealthViewSet(viewsets.GenericViewSet):
    """健康检查 — 无需认证"""
    authentication_classes = []
    permission_classes = []

    def list(self, request):
        checks = {
            'status': 'ok',
            'database': self._check_db(),
            'redis': self._check_redis(),
            'timestamp': timezone.now().isoformat(),
        }
        all_ok = all(
            v == 'ok' for v in checks.values() if isinstance(v, str)
        )
        status = 200 if all_ok else 503
        return Response(checks, status=status)

    def _check_db(self):
        try:
            connections['default'].cursor()
            return 'ok'
        except Exception as e:
            return str(e)

    def _check_redis(self):
        try:
            r = get_redis_connection('default')
            r.ping()
            return 'ok'
        except Exception as e:
            return str(e)
```

### 步骤 3：请求耗时统计中间件

```python
# apps/common/middleware.py
import time
import logging

logger = logging.getLogger('performance')

class RequestMetricsMiddleware:
    """记录每个请求的耗时"""

    def __init__(self, get_response):
        self.get_response = get_response

    def __call__(self, request):
        start = time.monotonic()
        response = self.get_response(request)
        duration_ms = (time.monotonic() - start) * 1000

        if duration_ms > 1000:  # 慢请求告警
            logger.warning(
                "Slow request",
                extra={
                    'path': request.path,
                    'method': request.method,
                    'duration_ms': round(duration_ms, 2),
                    'status': response.status_code,
                },
            )

        response['X-Response-Time'] = str(round(duration_ms, 2))
        return response
```

### 步骤 4：关键监控指标

| 指标 | 工具 | 告警阈值 |
|------|------|---------|
| API 响应时间 | 中间件 + JSON日志 | P95 > 1s |
| 错误率 | Sentry | 5xx > 1% |
| 数据库慢查询 | Django `connection.queries` | > 100ms |
| Celery 任务积压 | Flower / Prometheus | > 100 pending |
| 磁盘空间 | 系统监控 | < 10% free |
| 内存使用 | 系统监控 | > 80% |

## 日志级别使用规范

| 级别 | 场景 | 示例 |
|------|------|------|
| DEBUG | 开发调试细节 | SQL 查询、中间变量 |
| INFO | 关键业务流程 | 用户创建、角色变更 |
| WARNING | 可恢复的异常 | 慢请求、缓存穿透 |
| ERROR | 需人工介入 | 支付失败、第三方API不可用 |
| CRITICAL | 系统级故障 | 数据库不可用、磁盘满 |

# 15 - 前端安全与优化

## 适用场景
页面开发完成后，补充安全防护和性能优化。

## 关联原则
- 原则5：输入验证与输出转义 — XSS 防护是最后一道防线
- 原则6：最小权限 — 前端权限控制只是 UI 层，后端才是真正的防线

## 核心规则
1. **XSS 防护**：`v-text` 代替 `{{ }}`，`v-html` 必须经过 DOMPurify
2. **Token 存储**：不存 localStorage（用 httpOnly cookie），至少存 sessionStorage
3. **路由守卫**：每个路由检查权限
4. **打包优化**：路由懒加载、Tree Shaking、gzip

## 开发步骤

### 步骤 1：XSS 防护

文件：`front-end/src/security/sanitize.ts`

```typescript
import DOMPurify from 'dompurify'

/** 净化用户输入的 HTML（防止 XSS） */
export function sanitizeHtml(dirty: string): string {
  return DOMPurify.sanitize(dirty, {
    ALLOWED_TAGS: ['b', 'i', 'em', 'strong', 'a', 'p', 'br', 'ul', 'ol', 'li'],
    ALLOWED_ATTR: ['href', 'target', 'rel'],
  })
}

/** 纯文本转义（用于 v-text 以外的场景） */
export function escapeHtml(text: string): string {
  const map: Record<string, string> = {
    '&': '&amp;', '<': '&lt;', '>': '&gt;',
    '"': '&quot;', "'": '&#x27;',
  }
  return text.replace(/[&<>"']/g, (c) => map[c])
}

/** 创建安全的 v-html 内容 */
export function safeHtml(html: string): string {
  return sanitizeHtml(html)
}
```

使用方式：
```vue
<template>
  <!-- ✅ 安全：纯文本 -->
  <div v-text="userInput"></div>

  <!-- ✅ 安全：净化后的 HTML -->
  <div v-html="sanitized"></div>

  <!-- ❌ 危险：直接渲染用户输入 -->
  <!-- <div v-html="userInput"></div> -->
</template>

<script setup>
import { computed } from 'vue'
import { safeHtml } from '@/security/sanitize'
const sanitized = computed(() => safeHtml(props.userInput))
</script>
```

### 步骤 2：Session 认证安全

Session认证下，认证状态由后端维护，前端无需手动管理Token。需要注意的安全点：

```typescript
// front-end/src/api/client.ts — 关键安全配置
const client = axios.create({
    withCredentials: true,          // 携带 Cookie（Session ID）
    xsrfCookieName: 'csrftoken',    // Django CSRF cookie名
    xsrfHeaderName: 'X-CSRFToken',  // Django CSRF header名
})
```

**安全要点**：
- Session ID 存储在 httpOnly Cookie 中，JavaScript 无法读取（防XSS窃取）
- CSRF Token 由 Django 中间件自动管理，前端 Axios 通过 `xsrfCookieName`/`xsrfHeaderName` 自动处理
- 登出时调用 `POST /api/auth/logout/`，后端清除 Session

```typescript
// front-end/src/stores/user.ts — 登出 action
async logout({ commit }) {
    await client.post('/auth/logout/')
    commit('CLEAR')
}
```

## 常见前端安全漏洞与修复

| 漏洞 | 成因 | 修复 |
|------|------|------|
| XSS | `v-html` 未净化 | DOMPurify.sanitize() |
| CSRF | 未配置CSRF Token | Django CSRF中间件 + Axios xsrfCookieName |
| Session 劫持 | Cookie未设httpOnly | Django SESSION_COOKIE_HTTPONLY=True |
| 敏感信息暴露 | console.log 打印敏感数据 | 生产环境 strip console |
| 依赖漏洞 | 未更新的 npm 包 | `npm audit` + 定期更新 |

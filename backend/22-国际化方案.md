# 22 - 国际化方案

## 适用场景
管理系统需要支持多语言（中/英），包括后端错误消息、Model verbose_name、前端UI文案。

## 关联原则
- 原则5：输入验证 — 错误消息国际化
- 原则3：关注点分离 — 文案与代码分离

## 核心规则
1. **后端用 Django i18n**：`gettext` / `gettext_lazy`
2. **前端用 vue-i18n**：JSON 语言包
3. **错误码消息支持多语言**（ErrorCode 枚举 + 翻译文件）
4. **语言切换不影响数据**（数据库存储原始值，仅展示层翻译）

## 开发步骤

### 步骤 1：Django 后端 i18n 配置

文件：`config/settings/base.py` — 追加 i18n 配置

```python
from django.utils.translation import gettext_lazy as _

LANGUAGE_CODE = 'zh-hans'
LANGUAGES = [
    ('zh-hans', _('简体中文')),
    ('en', _('English')),
]
LOCALE_PATHS = [BASE_DIR / 'locale']
USE_I18N = True
USE_L10N = True
```

### 步骤 2：Model verbose_name 国际化

```python
from django.utils.translation import gettext_lazy as _

class User(BaseModel):
    username = models.CharField(max_length=150, verbose_name=_("用户名"))
    email = models.EmailField(verbose_name=_("邮箱"))

    class Meta:
        verbose_name = _("用户")
        verbose_name_plural = _("用户")
```

### 步骤 3：编译翻译文件

```bash
# 生成 .po 模板
python manage.py makemessages -l zh_hans
python manage.py makemessages -l en

# 编辑 locale/zh_hans/LC_MESSAGES/django.po 和 locale/en/LC_MESSAGES/django.po

# 编译 .po → .mo
python manage.py compilemessages
```

### 步骤 4：ErrorCode 多语言支持

文件：`apps/common/exceptions.py` — 扩展

```python
from django.utils.translation import gettext as _

class ErrorCode(Enum):
    ROLE_NOT_FOUND = ErrorDetail("ROLE_NOT_FOUND", _("角色不存在"), 404)
    # ... 使用 gettext_lazy 延迟翻译
```

### 步骤 5：前端 vue-i18n 配置

```bash
npm install vue-i18n@9
```

文件：`front-end/src/i18n/index.ts`

```typescript
import { createI18n } from 'vue-i18n'
import zh from './locales/zh-CN.json'
import en from './locales/en-US.json'

const i18n = createI18n({
    legacy: false,
    locale: localStorage.getItem('lang') || 'zh-CN',
    fallbackLocale: 'zh-CN',
    messages: { 'zh-CN': zh, 'en-US': en },
})

export default i18n
```

### 步骤 6：语言切换组件

```vue
<template>
    <a-select v-model="currentLang" @change="switchLang" size="small">
        <a-option value="zh-CN">中文</a-option>
        <a-option value="en-US">English</a-option>
    </a-select>
</template>

<script setup>
import { ref } from 'vue'
import { useI18n } from 'vue-i18n'

const { locale } = useI18n()
const currentLang = ref(locale.value)

function switchLang(lang: string) {
    locale.value = lang
    localStorage.setItem('lang', lang)
}
</script>
```

## 翻译文件管理策略

| 类型 | 管理方式 | 工具 |
|------|---------|------|
| 后端 Model 文案 | `.po` / `.mo` 文件 | `makemessages` / `compilemessages` |
| 后端 ErrorCode 消息 | `.po` 文件（与上合并） | Django i18n |
| 前端 UI 文案 | JSON 语言包 | vue-i18n |
| 前端错误码映射 | 从后端自动生成 | `export_error_codes` 命令 |

# 07 - 权限与安全开发

## 适用场景
RBAC 权限系统实现、数据权限控制、API 访问控制。

## 关联原则
- 原则6：最小权限 — 每个 action 只给必要的权限
- 原则5：输入验证 — 白名单验证 + 转义

## 核心规则
1. **权限检查在 ViewSet 层**（不在 Service 层）
2. **DRF 的 `BasePermission` 实现自定义权限类**
3. **数据权限用 `get_queryset` 过滤**（用户只能看自己的数据）
4. **数据库账户分离**（应用账户 vs 开发账户）

## 开发步骤

### 步骤 1：创建权限定义表（见 Model 层）

已完成 `Permission`、`Role`、`RolePermission` 模型。

### 步骤 2：创建 DRF 权限类

文件：`apps/rbac/permissions/user_permissions.py`

```python
from rest_framework.permissions import BasePermission

class IsSelfOrAdmin(BasePermission):
    """本人或管理员可操作"""

    def has_object_permission(self, request, view, obj):
        if request.user.is_superuser:
            return True
        return obj.id == request.user.id

class HasPermissionCode(BasePermission):
    """按权限编码控制访问"""

    def __init__(self, *codes: str):
        self.required_codes = codes

    def has_permission(self, request, view):
        if not request.user or not request.user.is_authenticated:
            return False
        if request.user.is_superuser:
            return True

        user_permissions = request.user.get_all_permission_codes()
        return all(code in user_permissions for code in self.required_codes)

class IsOwner(BasePermission):
    """数据所有者 - 仅能操作自己的数据"""

    def has_object_permission(self, request, view, obj):
        return getattr(obj, 'user_id', getattr(obj, 'owner_id', None)) == request.user.id
```

### 步骤 3：数据权限 — 按角色过滤查询

```python
class UserViewSet(viewsets.GenericViewSet):
    
    def get_queryset(self):
        """数据权限：普通用户只看自己，管理员看全部"""
        if self.request.user.is_superuser:
            return User.objects.all()
        return User.objects.filter(department_id=self.request.user.department_id)
```

### 步骤 4：安全配置模板（settings）

```python
# config/settings/prod.py - 安全配置

# HTTPS 强制
SECURE_SSL_REDIRECT = True
SESSION_COOKIE_SECURE = True
CSRF_COOKIE_SECURE = True

# HSTS
SECURE_HSTS_SECONDS = 31536000
SECURE_HSTS_INCLUDE_SUBDOMAINS = True

# 防点击劫持
X_FRAME_OPTIONS = 'DENY'

# 内容安全策略
SECURE_CONTENT_TYPE_NOSNIFF = True

# CORS - 精确控制（不是 *）
CORS_ALLOWED_ORIGINS = ['https://admin.company.com']
CORS_ALLOW_METHODS = ['GET', 'POST', 'PUT', 'PATCH', 'DELETE']
CORS_ALLOW_HEADERS = ['Content-Type', 'Authorization']
```

### 步骤 5：数据库权限（运维层面）

```sql
-- 应用账户：最小权限
CREATE USER app_user WITH PASSWORD 'xxx';
GRANT SELECT, INSERT, UPDATE, DELETE ON ALL TABLES IN SCHEMA public TO app_user;

-- 禁止应用账户 DDL
REVOKE CREATE ON SCHEMA public FROM app_user;

-- 迁移账户：仅 CI/CD 使用
CREATE USER migration_user WITH PASSWORD 'xxx';
GRANT ALL ON SCHEMA public TO migration_user;
```

## 常用权限类型速查
| 类型 | 适用场景 | 实现 |
|------|---------|------|
| 接口权限 | API 允许谁调用 | DRF `permission_classes` |
| 对象权限 | 能否操作某条记录 | `has_object_permission` |
| 数据权限 | 查询结果过滤 | `get_queryset` 重写 |
| 字段权限 | 某些字段仅特定角色可见 | Serializer 动态字段 |
| 菜单权限 | 前端菜单可见性 | 后端返回用户菜单树 |

## 反模式（禁止）
- ❌ `ALLOWED_HOSTS = ['*']` → 生产环境必须指定
- ❌ `CORS_ALLOW_ALL_ORIGINS = True` → 生产环境必须指定
- ❌ 权限逻辑写在 Service 层
- ❌ 数据库用 root 账号跑应用

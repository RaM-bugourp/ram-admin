# 19 - 数据导出与报表

## 适用场景
管理后台列表数据导出为 Excel/CSV，数据统计图表生成。

## 关联原则
- 原则5：输入验证 — 导出参数校验（时间范围、数据量上限）
- 原则6：最小权限 — 导出权限独立控制

## 核心规则
1. **导出权限独立**（`export:user` 不等于 `user:read`）
2. **数据量上限**（防止单次导出拖垮数据库）
3. **异步导出**（超过阈值自动转为后台任务）
4. **CSV 优先于 Excel**（流式处理，内存友好）

## 开发步骤

### 步骤 1：Django Export Mixin

文件：`apps/common/export.py`

```python
import csv
from io import StringIO
from django.http import HttpResponse, StreamingHttpResponse
from django.db.models import QuerySet


class Echo:
    """CSV 流式写入缓冲"""
    def write(self, value):
        return value


class ExportMixin:
    """ViewSet 混入 — 提供 GET ?_export=csv 导出功能"""
    export_fields = []       # 子类覆盖：导出的字段列表
    export_headers = []      # 子类覆盖：CSV 表头（中文）
    export_max_rows = 50000  # 单次导出上限
    export_chunk_size = 1000 # 流式分批大小

    def list(self, request, *args, **kwargs):
        export_format = request.query_params.get('_export')
        if export_format and export_format.lower() == 'csv':
            return self._export_csv()
        return super().list(request, *args, **kwargs)

    def _export_csv(self):
        """流式导出 CSV"""
        queryset = self.get_queryset()[:self.export_max_rows]

        # 获取字段值（支持嵌套字段如 'department__name'）
        def get_field_values(queryset):
            for obj in queryset.iterator(chunk_size=self.export_chunk_size):
                row = []
                for field in self.export_fields:
                    value = obj
                    for part in field.split('__'):
                        value = getattr(value, part, '')
                    row.append(str(value) if value is not None else '')
                yield row

        buffer = Echo()
        writer = csv.writer(buffer)

        # 流式响应
        response = StreamingHttpResponse(
            streaming_content=(
                writer.writerow(row) for row in self._iter_rows(writer, queryset)
            ),
            content_type='text/csv; charset=utf-8-sig',
        )
        response['Content-Disposition'] = 'attachment; filename="export.csv"'
        return response

    def _iter_rows(self, writer, queryset):
        """生成器：先写表头，再写数据"""
        yield writer.writerow(self.export_headers)
        for obj in queryset.iterator(chunk_size=self.export_chunk_size):
            row = [
                str(self._get_nested_attr(obj, field) or '')
                for field in self.export_fields
            ]
            yield writer.writerow(row)

    @staticmethod
    def _get_nested_attr(obj, field_path: str):
        value = obj
        for part in field_path.split('__'):
            value = getattr(value, part, None)
            if value is None:
                return ''
        return value
```

### 步骤 2：ViewSet 使用示例

```python
class UserViewSet(ExportMixin, viewsets.GenericViewSet):
    queryset = User.objects.active()
    export_fields = [
        'id', 'username', 'email', 'is_active',
        'department__name', 'created_at',
    ]
    export_headers = [
        'ID', '用户名', '邮箱', '是否激活',
        '部门', '创建时间',
    ]
    export_max_rows = 10000

    # 导出权限独立于查看权限
    permission_classes_by_action = {
        'list': [HasPermissionCode('user:read')],
    }

    def get_permissions(self):
        if self.request.query_params.get('_export'):
            return [HasPermissionCode('export:user')()]
        return super().get_permissions()
```

### 步骤 3：异步导出（大数据量）

```python
# apps/common/tasks.py
from celery import shared_task
from django.core.mail import EmailMessage

@shared_task
def async_export_csv(
    model_path: str, fields: list, headers: list,
    filters: dict, user_email: str,
):
    """异步导出任务 — Celery 后台执行"""
    from django.apps import apps
    model = apps.get_model(model_path)

    queryset = model.objects.filter(**filters)
    csv_content = StringIO()
    writer = csv.writer(csv_content)
    writer.writerow(headers)

    for obj in queryset.iterator(chunk_size=2000):
        row = [str(getattr(obj, f, '')) for f in fields]
        writer.writerow(row)

    # 邮件发送
    email = EmailMessage(
        subject='数据导出完成',
        body='请查收附件',
        to=[user_email],
    )
    email.attach('export.csv', csv_content.getvalue(), 'text/csv')
    email.send()
```

### 步骤 4：前端导出按钮

```vue
<template>
    <a-button @click="handleExport" :loading="exporting">
        导出CSV
    </a-button>
</template>

<script setup>
import { ref } from 'vue'
import client from '@/api/client'
import { Message } from '@arco-design/web-vue'

const exporting = ref(false)

async function handleExport() {
    exporting.value = true
    try {
        const response = await client.get('/users/', {
            params: { _export: 'csv', ...filterParams },
            responseType: 'blob',
        })
        const url = URL.createObjectURL(response)
        const a = document.createElement('a')
        a.href = url
        a.download = `用户列表_${new Date().toISOString().slice(0, 10)}.csv`
        a.click()
        URL.revokeObjectURL(url)
    } catch {
        Message.error('导出失败')
    } finally {
        exporting.value = false
    }
}
</script>
```

## 导出安全规则

| 规则 | 原因 |
|------|------|
| 导出权限独立（`export:user`） | 能看列表不等于能导出数据 |
| 单次导出上限 10000 行 | 防止拖库 |
| 超过 5000 行转异步任务 | 防止请求超时 |
| 导出字段白名单 | 防止暴漏敏感字段 |
| 不导出 password_hash 等字段 | 即使字段在白名单也强制排除 |

## 反模式（禁止）
- ❌ 一次性加载全部数据到内存 → 用 `iterator(chunk_size=XX)` 流式
- ❌ 导出权限复用查看权限 → 独立的 `export:xxx` 权限
- ❌ 导出包含敏感字段 → 导出字段白名单
